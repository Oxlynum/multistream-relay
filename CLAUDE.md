## What it is
**SlimCast** ‚Äî consumer multistreaming SaaS. OBS pushes one HEVC stream ‚Äî from **any HEVC-capable hardware encoder** (Apple VideoToolbox on Mac; NVENC / AMD AMF / Intel QSV on PC) ‚Äî over SRT to a trusted VPS hub; the hub bridges it to a cloud GPU (HEVC-decode ‚Üí H.264-encode) and fans the returned video out to Twitch, Kick, YouTube, TikTok. Stream keys never reach the GPU. **Mac and PC;** nothing requires a terminal.

**Business model:** two-tier (Phase 3, behind `SLIMCAST_BILLING_ACTIVE`, default OFF). **PAYG:** buy tokens (1 token = 1 hr base transcode = $2); passthrough 0.1 tok/hr, transcode 1.0 + adders. **Subscription $20/mo:** monthly token allotment (15, rolls over capped at 30) + passthrough 0.05 tok/hr (cheaper but **never free** ‚Äî a 24/7 idle passthrough still burns VPS bandwidth). Spendable = allotment (spent first) + purchased balance. 2 free tokens on signup. Kill-on-empty is **uniform** across plans (stream stops when spendable ‚â§ 0). Full model: ¬ß9 / ¬ß9a below.

## Architecture (load-bearing ‚Äî read before changing)

1. **OBS ‚Üí hub: HEVC over SRT (UDP); hub ‚Üí GPU: mpegts-over-TLS (TCP) on :8899.** OBS publishes only to the trusted VPS hub (`srt://<hub>:8890?streamid=publish:<key>`) ‚Äî never to a GPU. The GPU's only ingress is the hub's TCP bridge on :8899 (no SRT/UDP into the GPU); it HEVC-decodes ‚Üí H.264-encodes and returns RTMPS video to the hub, which pushes to every platform. **Stream keys never reach the GPU.** No RTMP ingest. The TCP bridge lifts the old UDP cage ‚Äî RunPod is a first-class GPU backend again. RTMP `:1935` binds on the hub/GPU only as a readiness beacon (TCP-provable from serverless); OBS never publishes to it.

2. **Internal loopback: SRT, not RTSP.** MediaMTX runs one SRT server on `:8890` ‚Äî OBS publishes externally, FFmpeg reads loopback (`srt://127.0.0.1:8890?streamid=read:<key>`). RTSP mangles temporal-layered HEVC (Apple VideoToolbox, and NVENC/AMF with a B-pyramid) ‚Üí dropped frames. Do not switch to RTSP.

3. **FFmpeg pinned to jellyfin-ffmpeg 7.1.4-3.** Do not change to chase NVENC failures on driver ‚â•570 hosts ‚Äî that's a host driver bug (NVENC API backward-compat means no FFmpeg version fixes it). The preferenceTier + boot self-test handle it.

4. **Hardware codecs only.** NVDEC decode + NVENC H.264 encode. CPU only for portrait crop+scale. YouTube landscape = `-c copy` HEVC passthrough.

5. **Vercel stores config, GPU executes it.** Stream keys in Supabase; agent polls Vercel every 10s. One `SLIMCAST_API_KEY` env var on the pod.

6. **OBS plugin talks to Vercel, not the GPU.** GPU IP never reaches the plugin.

7. **Grouped transcode + tee fan-out (split GPU/hub since 2026-06-29).** The GPU does one shared H.264 encode per orientation (`build_gpu_transcode_cmd`) and returns it to the hub; the hub tees each return to every platform in that group (`build_deliver_cmd`) plus HEVC passthrough for YouTube (`build_passthrough_cmd`). `onfail=ignore` ‚Äî one platform drop doesn't affect others. `use_fifo=1` on each tee output ‚Äî each RTMP destination runs in its own thread with a 512-frame queue so TCP backpressure on one platform (e.g. Twitch slowing ACKs) cannot stall the encoder. Platforms in the same group share one bitrate. Do not revert to per-platform transcodes. (The old all-in-one `build_group_cmd` ‚Äî transcode+tee in one pod ‚Äî was deleted.)

8. **GPU chosen by broker, never by hand.** Ranked: `preferenceTier` (NVENC driver-regression soft-demote) ‚Üí **haversine distance to the HUB** (the VPS‚ÜîGPU bridge leg is the dominant latency cost ‚Äî NOT user distance) ‚Üí price. Hard $1/hr ceiling. **The sole provision path is the GPU-backend race in `lib/vps-broker.ts`** (`startGpuBackendRace`/`reraceGpuBackend`, N=1), using the shared `rankedCandidates`/`startProvisionRace` (push-readiness: pods self-report via `POST /api/agent/ready` CAS-winner / `POST /api/agent/failed` kick-next-round). Bad hosts known in ~1s (agent exits + POSTs /failed) instead of a probe timeout. `gpu-broker.ts` lost the v1 synchronous cascade (`provisionGpu`/`probeRtmp`/`waitForIp`) and the `mode:'all-in-one'|'backend'` param (gone everywhere).
   - Registry: `lib/providers/index.ts` ‚Äî one role-uniform `ACTIVE_GPU_PROVIDERS` (`= [vast, runpod]`); `getProvider()`/`resolveProvider()`.
   - GPU filter (`vast.ts`/`runpod.ts` `listCandidates()`): Turing+ (`compute_cap >= 750`, the NVENC+NVDEC HEVC‚ÜíH.264 proxy), consumer‚Üíwhole-GPU (`gpu_frac >= 1.0`; datacenter/MIG ‚Üí fractional OK), a bandwidth sanity floor, all-in price ‚â§ ceiling. **Removed: the SRT-era `‚â•3 direct ports`/UDP-port screens** ‚Äî a GPU needs only the TCP bridge port (8899, free from `EXPOSE`). Pool is now ~all NVENC/NVDEC cards (incl. fractional datacenter cards).
   - **RunPod re-added** as a first-class GPU backend (the TCP bridge lifts the UDP cage that had banned it). Vultr planned next.
   - Broker knobs: `lib/datacenters.ts` (`PRICE_CEILING`, readiness timeouts, `FALLBACK_LAT/LON`).

9. **Billing on heartbeat (two-tier, Phase 3).** Single source of truth = `lib/billing.ts` `billingLineItems()` (feeds BOTH `computeBurnRate` deduction AND `buildPricingBreakdown` UI ‚Äî they can't drift). **Passthrough group** (YouTube HLS + eligible-Twitch eRTMP + future): ONE flat charge, not per-platform ‚Äî 0.05 tok/hr subscriber / 0.1 PAYG. **Transcode:** base 1.0 + 0.2/extra landscape + 0.2/portrait-different + 0.1/portrait-dual + 0.5 1440p (`has_2k_addon`, suppressed while throttled below 1440p) + 0.5 if >3 NVENC. **Deduction** = `lib/billing-clock.ts` `billStreamInterval()` ‚Üí `deduct_tokens` RPC (allotment-first, then purchased, `FOR UPDATE` row lock). The deducting clock is the VPS hub **Clock A** (`handleVpsStatus`, per-tenant, batch-loaded); the GPU-backend heartbeat is telemetry only (the old `label='pod'` all-in-one billing clock was deleted with the direct path 2026-06-29); dashboard/dock polls never deduct. All gated on `SLIMCAST_BILLING_ACTIVE` (default OFF = free). Kill-on-empty when `spendable ‚â§ 0` (allotment+purchased). See `lib/nvenc-utils.ts` `requiredNvencSessions()`.

9a. **Budget throttle: degrade, don't kill. ‚Äî DEFERRED 2026-06-29:** the margin-throttle (`relay/budget.py` `BudgetController` + `throttle_config`) was REMOVED with the all-in-one path; re-introduction on the hub (whose egress is the real cost lever) is a future VPS/billing-plan item ‚Äî billing is OFF now, so safe. `CostMeter` is KEPT (the GPU backend uses it for bridge cost telemetry). The mechanism below is retained as the design-of-record for hub re-introduction. The flat user price hides a variable Vast bill (GPU + egress + ingress $/TB). The $1/hr broker ceiling is a provision-time *estimate*; live spend is uncapped (OBS source bitrate and YouTube HEVC passthrough follow whatever OBS sends). The pod closes that loop: `relay/budget.py` `CostMeter` reads `/proc/net/dev` each heartbeat (excludes `lo` ‚Äî the SRT loopback isn't billed) and computes real $/hr from cost rates injected at provision (`SLIMCAST_GPU_RATE_USD`, `SLIMCAST_EGRESS_USD_PER_TB`, `SLIMCAST_INGRESS_USD_PER_TB`, `SLIMCAST_COST_CEILING_USD`). `BudgetController` maps cost ‚Üí a quality tier (discrete ladder in `TIERS`) with **down-fast/up-slow hysteresis** (>ceiling ‚Üí throttle a step; <85% for 3 beats ‚Üí recover; 85‚Äì100% dead-band ‚Äî prevents FFmpeg-restart flapping). Three levers, applied together per tier: (a) transcode bitrate caps + (b) resolution downscale (`scale_cuda` landscape / portrait scale) via `throttle_config` ‚Üí `sup.apply()` (pod-local, ~0s); (c) **OBS source bitrate** ‚Äî the only lever touching ingress + YouTube passthrough ‚Äî reported as `suggested_ingest_kbps` in the heartbeat ‚Üí surfaced in `/api/gpu/status` ‚Üí plugin calls `obs_encoder_update()` on the live encoder (~15s end-to-end). Ceiling is **$1.00/hr for `has_2k_addon`, else $0.50**. Floor tier = the user's entitled resolution (controller never recovers above it). `SOURCE_WIDTH/HEIGHT` are set at provision from the user's max output resolution so the downscale guard knows the true source. This protects SlimCast's **margin**, not the user's bill ‚Äî the user pays burn_rate regardless. **YouTube stays HEVC passthrough** (best quality/bit); never transcode it to "save" egress ‚Äî the OBS source lever already caps it. Dock shows a calm "Live ¬∑ quality auto-adjusted" when throttled.

10. **Pod safety: defense-in-depth.** A rogue pod is the #1 financial risk.
    - Atomic provision claim: row reserved before pod created ‚Üí 409 on race. Requires saved card + credits.
    - Plugin: Stop ‚Üí destroy; orphan auto-destroy after ~10s idle (catches OBS crash).
    - Heartbeat self-destruct: credits‚â§0, idle>5m (`idle_since`), or past `max_session_at`.
    - Agent watchdogs: stop outputs after ~60s missed heartbeats; idle/dead boxes are reaped by the universal-lease sweep + cron reaper (the pod `/api/agent/terminate` route was deleted 2026-06-29; the GPU self-destructs via the `/failed`/loser-CAS path, the hub via `teardownHub`).
    - Cron reaper (`/api/cron/reap`, daily): stale >150s, never-paired >180s, past-deadline, idle. Reconciles against all providers to catch true orphans. `CRON_SECRET` optional.
    - All teardown via `lib/pod-teardown.ts` `teardownInstance()` (idempotent: destroy + revoke key + delete row + close stream_session).

11. **Stream keys never reach the public.**
    - Encrypted at rest: AES-256-GCM in `platform_connections.stream_key_encrypted` (`lib/crypto.ts`). **Decrypted keys go ONLY to the trusted VPS hub** (`/api/agent/hub-config`); a rented GPU NEVER receives a stream key (it only transcodes and returns video). Dashboard queries exclude the key column.
    - The `:8080` FastAPI debug panel (`relay/app.py`) was **removed 2026-06-29** ‚Äî closes the internet-exposed-panel item; hub + GPU mirror FFmpeg stderr to `docker logs`.
    - Only `NEXT_PUBLIC_*` vars are public; service-role + Stripe + `STREAM_KEY_SECRET` are server-only.

12. **12h cap is confirmable, not a hard kill.** `max_session_at` = now+12h at provision. Final 30m: status returns `confirm_required:true`; dock shows countdown + confirm button. Confirming extends 12h. Unconfirmed ‚Üí heartbeat hard-kills ‚Üí reaper backstops.

13. **Money paths are idempotent + rate-limited.** `credit_payment_once(payment_id,‚Ä¶)` deduped by Stripe payment ID (no double-credit on webhook retry). Sensitive routes: `checkRateLimit()` (Supabase fixed-window, fails open). Checkout price is server-side.

14. **Relay image: driver-level only, not CUDA toolkit.** All GPU surface (NVENC, NVDEC, hwaccel) mounted from host by NVIDIA container runtime. Base: `nvidia/cuda:12.4.1-base-ubuntu22.04`. `NVIDIA_DRIVER_CAPABILITIES` must include `video` + `compute` (set in Dockerfile; Vast may not set them implicitly). Rollback: `docker buildx imagetools create` to retag a known-good `:<sha>` to `:latest` (~2s).

Eligibility is probed on Twitch key save / OAuth connect / manual re-check and stored on `platform_connections` (`twitch_hevc_eligible`, `twitch_use_passthrough`, `twitch_max_height`, `twitch_eligibility_checked_at`). `agent-config.ts` routes Twitch to `mode:'ertmp'` ONLY when `twitch_hevc_eligible && twitch_use_passthrough`; otherwise it falls through to the H.264 landscape tee group. Dashboard surfaces the passthrough toggle + 2K only when eligible. The full eRTMP relay stack (GPU spoof, BPM SEI `relay/bpm_inject.py`, `-rtmp_enhanced_codecs hvc1`) is correct and lights up automatically for any eligible channel.
